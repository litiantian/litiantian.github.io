前后台交互

今天小姐姐不分享代码，来谈谈我对前后台交互的认知。

在web开发中，前后台交互是一个至关重要的环节，前后台需要及时沟通，工作才能顺利进行，有时候就是因为沟通不够，然后造成许多白用功。不过，对于刚接触前端开发的小白来说，前后端交互的概念很模糊，不过不必紧张，经过几个项目之后，就会好的。

php + 前后台交互

首先，我们的明白服务器的概念

服务器

什么是服务器？分为四大块：

1.硬件：它是一台主机，基于三种主流的操作系统：windows、mac OS、UNIX/LINUX.

2.软件：支持服务，支持软件的
它的服务类型分为三种：

  • FTP:提供文件上传下载的服务器，可以称为文件服务器
  • HTPP/HTPPS:提供web服务的服务器，支持的软件有Ningx和Apache
  • mySQL:提供Mysql数据库服务的服务器

3.联网：服务器需要通过网络连接，为外界提供服务

  1. 后台语言环境的支持：如：js 、java、 php(使用最广泛的编程语言)、Python

web服务器的配置：首先建立网站根目录，使用后台语言编译。

前后台交互

一、 服务器端主要完成的功能
  1. 接受前台提交的数据
  2. 对前台提交的数据进行处理
  3. 服务器向前端返回数据
  4. 存储数据
二、数据传送方式
  1. get方式：
  • 不安全；
  • get数据有限制；
  • get是通过url提交数据?参数
  1. post方式：
  • post数据理论上没有限制
  • post数据是在http请求头的主体部分发送的
  • 相对安全
三、数据提交方式

1. 在链接地址上使用?提交 ， 只有get一种传送方式。

2. form表单方式

  • 可以实现POST与GET两种数据传送方式
  • 可以跨域提交数据
  • 使用iframe方式实现局部刷新（iframe）不支持跨域

3. 通过script标签的src属性

  • 只能使用get方式
  • 独立的实现局部刷新
  • 支持跨域
  • JSON与JSONP 他们的详细介绍在我的上一篇博客里已经说过，这里就不重复了，想要了解的可以点击：http://www.jianshu.com/p/f46dd756873e

4 .XHR对象，XMLHttpRequest

  • 创建对象： var xhr = new XMLHttpRequset();
  • 方法：xhr.open(); 三个参数(GET/POST,url,同步|异步)
  • 事件：onreadyStateChange: 当XHR状态发生变化的时候会执行
    onload: 数据接收完毕互执行
  • 属性：responseText : 服务器返回的数据
    readyState XHR的执行状态，有五种状态：

    0 - 初识状态
    1 - 创建链接
    2 - 发送数据
    3 - 服务器开始返回数据
    4 - 数据接受完毕

5. FormData 模拟表单提交
创建表单对象：var formData = new FromData();
添加数据方法： append

6. jquery Ajax封装
常用的四种方法：

  1. $.ajax()
  2. $load()
  3. $get()
  4. post()
    具体用法大家可以查手册，这里就不过多介绍了。
