php文件上传

####一.首先看看html的form表单结构
form表单需要定义method的属性的值，可以为2种：get和post;
get是把参数列表加到对应的form表单的URL中，传输文件大小受限制，安全性低；post的传输是我们看不见的过程，安全性高，传输大小不受限制。
注意：一定要加上enctype属性，文件才能上传

二. php $_FILES

  1. $_FILES 和 $_POST、 $_GET一样都是数组，当获取form表单中文本信息时，使用form表单所对应的mathod的值($_POST和 $_GET)；当获取file文件时使用$_FILES。
  2. $_FILES的参数
  • name 上传的原文件名，不代表是文件，仅仅是字符串
  • tmp_name 文件上传到服务器的临时的文件名，上传后的文件位于该位置，这个是文件
  • type 上传的我文件类型
  • size 上传的文件大小
  • error 上传的文件错误信息

    三.文件拷贝与保存

  1. copy() 文件上传后，需要将临时文件copy到永久存放位置，否则程序结束后，会自动删除该文件;
  2. move_uploaded_file() 只拷贝上传的文件 安全性高

四.应用小例子

做个菜单管理后台；
需求：可以在管理页面上上传菜谱的照片，设定菜谱，及价格，并且记录到menu.txt文件上

  1. 由于步骤较多，我们可以把每个模块封装成对应的函数

  2. 点击提交时，调用函数实现功能

3.form的表单结构
注意看嵌套php的部分，使用foreach遍历保存数据的数组$menuList,将数组中的图片路径，菜名，价格等信息赋值给HTML标签

函数闭包

函数的闭包是个老生常谈的话题，但说起闭包，还是很难透彻的理解它的含义，在很多应用中都要使用闭包实现，要想搞明白函数闭包是什么，首先必须得明白变量的作用域。

变量的作用域

变量分为：全局变量和局部变量
1.局部变量：在函数内部声明的，不可以在函数外部使用。（本质原因：函数再被调用的时候会入栈，并且分配空间，函数内部的变量，都是在这块空间内部声明的，当函数运行完毕之后就会出栈，系统回收空间，函数内部的变量所占空间也会被回收，所以不能在函数外面使用函数内部定义的变量）
2.全局变量：在函数以外声明的。
eg1. 数组里面的两个a是同一个a。对象引用


eg2.函数t运行两次，就会有2个a。这两个a不是同一个a


eg3.执行同一次函数的时候，变量是相同的


上面三个例子充分解释了变量作用域的区别。

函数的闭包

闭包简单来说就是函数内部的函数，闭包里面使用局部变量和函数参数的时候，用的就是外部函数的局部变量。

1.同一个函数调用两次，会返回两个函数f1，和f2。 f1和f2里面的a分别是第一次和第二次调用fn的时候生成的a。所以是两个不同的a


2.调用一次函数返回两个函数f1，和f2。f1和f2里面的a都是第一次调用fn的时候生成的，所以是同一个a


闭包虽然用处很大，但它的弊端也不容忽视。闭包会导致函数运行结束之后局部变量的空间不会被释放从而造成内存泄露（执行的次数越多，占用的内存越多），所以一些经常需要重复执行的代码不适合使用闭包。

flex

flex弹性盒子是一种布局的方式，它能实现很多传统布局难以实现的特殊布局。

基于flex容器上设置的属性：

  1. 设置容器为弹性盒子

  2. flex-direction属性改变flex容器的主轴方向:默认值为row

    他有四个属性：flex-direction:row row-reverse column column-reverse

    * row:定义主轴方向为x轴，从左到右排列
* row-reverse: 定义主轴方向为x轴，从右到左排列
* column: 定义主轴方向为y轴，从上到下排列
* column-reverse: 定义主轴方向为xy轴，从下到上排列

  3. justify-content属性定义内容元素在主轴位置的对齐方式

    他有五个属性：flex-start flex-end center space-around space-between

    * flex-start:定义内容元素在主轴的起始位置对齐
* flex-end: 定义内容元素在主轴的结束位置对齐
* center：定义内容元素在主轴的中心位置对齐
* space-between：定义内容元素平均分配在主轴上，第一个元素在主轴开始的位置，最后一个元素在主轴结束位置
* space-around: 定义内容元素平均分配在主轴上，在每行上均匀分配弹性元素，相邻元素间距离相同，每行第一个元素到行首的距离为一半，最后一个元素到行尾的距离为一半

  4. algin-items定义元素在交叉轴上的对齐方式

    他有五个属性：flex-start flex-end center baseline stretch

    * flex-start：交叉轴的起点对齐。
* flex-end：交叉轴的终点对齐。
* center：交叉轴的中点对齐。
* baseline: 项目的第一行文字的基线对齐。
* stretch（默认值）：如果项目未设置高度或设为auto，将占满整个容器的高度。
  5. algin-content定义多行的对齐方式，当只有一行的时候没有效果
    algin-content: ;

  6. flex-wrap是否换行

    他有两个属性：wrap no-wrap

    *wrap 换行
*no-wrap 不换行默认
  7. flex-flow是flex-direction和flex-wrap的缩写
